# 问题意识驱动的设计推导（v2）

> v2 是 v1 的**姿态特化**——基础层、意识形态、原则形态、判据应用、反模式、失败补救等内容**全部沿用 v1**。v1 原文的哲学密度对 v2 已经足够。
>
> v2 补两类东西：(a) 把 v1 适用到"引导他人展开认知"场景时，姿态需要的调整（§一~三、§六~七）；(b) **深入浅出的落地手艺**——比喻↔技术细节的双向、每个抽象点必带具体例子、先把读者领进全局、能拔高成可决策的方向题就拔（§四），以及配套的失败模式"过度压缩=把展开成本甩给读者"（§五）。本文档不复述 v1，只在差异点标注 + 加 v2 特有的手艺与反模式。

# 仅 v2 需要的姿态调整

## 一、从审视转为邀请

v1 的预设场景是**审视**：已有的设计摆在面前，把它的原则梳理清楚——面向**结果**的整理。

v2 的预设场景是**邀请**：分析者带读者一起走一遍意识展开的过程，让读者在自己脑中复现"原则如何形成、设计如何被推出"——面向**过程**的同行。

骨架不变（意识 → 原则 → 推导）。区别在态度：v1 是已知答案的整理者，v2 是路途中的同行者。

## 二、为什么"邀请"必须是邀请

意识不是一个外在工具去解决问题——意识本身在分析的运动中**展开**。当原本的局部自洽被新材料逼到不安、必须重新组织时，**重新组织所依靠的"胚胎"在分析者自身意识里**——来自他的具体经验、训练、看过的同类对象。是 Husserl 意义上"已经在那里"的认知 horizon。

这就是 v2 不能"规定"的根本原因：**真正决定展开能否发生的是分析者意识里有没有那个胚胎**。Plato 的 anamnesis（回忆说）说知识不是被教会的、是被回忆起的；Heidegger 说真正的领会是已有"前理解"被新遭遇激活、不是被白纸状态地灌输；东亚思想里"道不可教，只可指"。三条传统都指向 v2 工作的同一个层级——**v2 只能创造让胚胎显现的情境，不能植入胚胎**。

**这件事在写作上的直接 implication**：意识陈述应优先**从观察到的具体现象直接归纳**得出。读者能在自己脑中复现"现象 → 意识"的推断时，他自己的胚胎才能被激活。从"在 X 约束下用 Y 手段达成 Z"这种抽象张力框架反推，邀请直接落空——读者拿到的是分析者的结论，不是自己的认知。

## 三、推导的展开姿态（v2 替换 v1 第三步的推导表）

v1 的推导是横向映射（设计 ↔ 原则）——回答"哪条设计推自哪条原则"，适合事后梳理已经收敛的设计。

v2 的推导是过程展示——读者读完之后能在自己脑中复现"原则如何叠加产生设计全貌"。形态由材料决定，不预设：可能让原则一条条加进来看每加一条产生什么新设计，可能让现象一个个出现看分析如何被它们逼向新原则，可能让分析的不安自然涌出看每次断裂如何带出新理解。

**不要给环路编号**。"Ring 1 / Ring 2 / 现在进入第三环"这种标签把内容变成排列好的展品，读者读到一半被结构提醒"你正在被结构性安排"，分析失去"被推动着走"的质感。**好的展开应让读者读完才意识到刚才走过了几个环**——形式从问题本身的动力组织，不被预设的章节切换规定。

## 四、深入浅出：把抽象落到读者能接住的具体

§二说邀请要让读者在自己脑中复现"现象 → 意识"。复现需要**落点**——光给抽象，读者无处挂靠。深入浅出就是给落点的手艺，五条：

**1. 方向是不对称的。** 你（作者）的思考方向是「具体技术细节 → 提炼成抽象 / 比喻」；你给读者的解释方向必须**反过来**：「比喻先行 → 一步步拆回真实细节 → 收成一个读者能拍的方向题」。读者沿你来时的反方向走回去，才能在自己脑里把细节重建出来。只给读者你提炼后的抽象结论，等于让他自己走完你走过的全程——他走不动。

**2. 每个抽象点后面必须紧跟一个落地的具体例子。** 任何"机制 X 有问题""层 Y 不一致"这种抽象判断，立刻跟一个"落到地上会怎样"的具体场景（谁、在什么设置下、哪一步、结果错成什么样）。**读者一问"具体指什么 / 这是啥意思 / 能不能举例"，不是他迟钝，是你漏了例子**——这是硬信号，不是可选润色。

**3. 先把读者领进全局（v1/v2 原本都缺的一环）。** 在"意识 → 原则 → 推导"开始之前，先用一段把读者领进整个来龙去脉：这件事在干嘛、走到哪了、为什么现在谈这个。v1/v2 都默认读者已在场景里——对一个**需要被领进来**理解全局的读者，这一步省不得，否则后面所有展开他都没有挂靠点。**先全局、后结构。**

**4. 能剥离语境、拔高成"读者当前就能答的方向题"，就直接问那个题。** 若一个技术细节能抽象成一个读者在已知上下文里就能拍的方向性决策，直接把那个决策题端给他——别逼他先消化全部技术细节才够得着决策。拔不动（决策真依赖全 context）才携带整个上下文逐步展开，**展开到完美解释整件事为止**。这与§二一致：拔高是为了让读者用**自己**的判断拍板，不是替他拍。

**5. 比喻要克制，且每个比喻必须【同步】带上真术语/真解释——不能空飘。** 比喻是给读者的临时挂靠点，不是终点。两条硬规则：(a) **数量克制**——比喻堆多了，读者得在多套隐喻之间反复换算，比直接给术语更累、更看不懂；一段里一个承重比喻足矣，其余直接上真名。(b) **同步落地**——每出现一个比喻，**当场**（不是"后面再拆"）就把对应的真实术语/机制并排写出来，比喻与真名同时出现，如"速记员（= distiller，压缩工具结果的那个模型）"、"记账（= 把每次 distill 决策 append 成一条 custom entry）"。判据：读者顺着你的比喻、却说不出它对应的真东西叫什么、在哪个文件 / 哪个机制，就是比喻空飘了。⚠️ 这条收紧 §四.1 的"比喻先行 → 一步步拆回"——"一步步"易被误读成"可以拖到后面才拆"；实际必须**同步**。

## 五、v2 特有的失败模式

v1 的反模式（意识缺位、原则伪装、推导跳跃、细节堆砌、意识与原则错位、事后合理化）全部沿用。v2 在邀请姿态下额外面临一类失败：**用方法替代思考**。

征兆：

- **术语密度高，剥掉术语后内容稀薄或重复**——校验：把所有术语换成大白话，剩下的话是否仍然有内容？
- **反馈来时不是深化已有理解，而是在已有结构旁加新模块/新表/新约束**——这是最难自我察觉的失败：每次加都觉得"为反馈服务"，合起来变成 bloat
- **给本来朴素的活动起学术名字**——如把"读者反馈调整"叫"用户画像分析"。添加方法论包装但没增加任何操作内容
- **把"我还在试错"包装成"我在邀请你展开"**——这是最深的故作深沉：方法成了掩盖"我没想清楚"的工具
- **过度比喻——比喻空飘、不落真术语**——以为比喻越多越亲民；实则读者要在多套隐喻之间反复换算，成本超过直接读术语，且脱了真名的比喻读完抓不住"到底指代码里哪个东西"。**"有时候比喻太多反而看不懂。"** 校验：通篇比喻，却数不出几个真术语 / 文件名 / 机制名。补救：砍冗余比喻，留一个承重比喻 + 每个比喻旁并排真名（见 §四.5）。
- **过度压缩——把展开成本甩给读者（假经济）**——以为"说得短 = 高效"，但压到读者得自己脑补、自己展开，他的理解反而更慢更累：**"说得太短，反而让读者理解太长。"** 真·深入浅出不是字少，是**让读者的理解即时发生**——该展开的地方全展开，不留"读者自行补全"的缺口。判据：读者读完若要反问"具体是什么 / 能举例吗 / 这啥意思"，就是你压过头了。⚠️ 与 v1 的"细节堆砌"和"匀质化"方向相反——那两条是"该少说处说太多"，这条是**"该多说处说太少"**：核心点本身欠展开。压缩 ≠ 凝练；凝练是密度高且读者一遍即懂，压缩是把理解工作量转嫁出去。

发生时：**放下方法，回到具体材料和具体问题**。补救不是更精细的方法，是更朴素的语言。

**v2 的成功本身没有内在判据**——读者声称"我懂了"不等于他真懂。Plato anamnesis 的老问题：苏格拉底说 boy 自己回忆出几何定理，但 boy 真的回忆的是同一个定理吗？还是他被 leading question 引导到了"看起来是同一个"的某个东西？v2 没法在内部解决这个问题——唯一的检验是读者能不能用所获得的认知做对接下来的事，这是外部的、滞后的、不可被 prompt 替代的检验。

承认这条边界比假装它不存在更接近真实的深度。

## 六、何时不用 v2

- **主体还在形成、连分析者自己都没想清楚**——还没东西可邀请。强行用 v2 会变成用方法替代思考
- **简单清晰的技术问题，直接答就行**——别把简单事情仪式化
- **读者明确表达"我只想要结论"**——不要把答案藏进过程
- **自己的分析正在用术语包装薄弱内容时**——方法已经变表演，停下检讨

## 七、和 v1 的选用

| 主体 | 选 |
|---|---|
| 已收敛设计的事后梳理 | v1 |
| 引导他人理解一个事物并产生独立洞察 | v2 |
| 自己设计要立结构 | v1 |
| 单层、扁平、原则少 | v1 |
| 多层认识在不同时刻形成、要展示这一过程 | v2 |
| 主体不清楚 / 还在共同探索阶段 | 都不用，先 trial-and-error |

最后一行最重要——**不是所有思考都该用方法。强行套方法是故作深沉的根源**。
